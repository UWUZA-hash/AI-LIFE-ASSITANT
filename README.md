# 生活助手 Skill — 项目说明

## 选题

**自拟题目**：AI 个人生活助手 — 任务排程 + 饮食管家联动

> 结合个人实际需求（学生日常排程 + 食堂饮食选择），设计一个跨域联动的 Hermes Agent Skill，让排程和饮食不再是两个独立功能，而是共享数据、作息联动、模式协调的统一生活管理工具。

---

## 功能简介

### Part A：任务排程

- **智能推断**：从任务关键词自动推断身份（学生/工作者）和动作类型（背/写/做/改/社交/运动）
- **置信度分叉**：>80%静默推断 / 50-80%问1个问题 / <50%坦诚询问
- **拖延克星**：连续移同任务→标记高拖延风险→自动预拆解为"穿上鞋"级别的小步骤
- **倒推开工日**：DDL - 历史耗时 - 拖延缓冲
- **持续学习**：耗时自修正、时段效率锁定/避开、忙时黑名单
- **能量时段感知**：记忆→早晨、深度→上午、创意→午后、体力→灵活

#### 🌟 排程两大核心亮点

**1. 拖延预拆解 — "穿上鞋"级别的启动门槛**

传统排程工具只会提醒"DDL快到了"，但拖延的人不是忘了，是启动不了。本 Skill 从行为数据中识别拖延模式：同一任务连续被移到"今天"≥3次 → 自动标记为高拖延风险 → 下次遇到同类任务直接预拆为 3-4 步"穿上鞋"级别子任务（如"打开文档→写第一段→存盘收工"），而不是等用户说"我做不到"才拆。

> 这不是规则触发，是 AI 从历史行为中发现的模式。没有 AI，这条链路根本不存在。

**2. 置信度分叉 — 不确定时只问 1 个问题**

用户说"搞定简历"，AI 判断难度=3、动作=写作、时段=上午，置信度 85% → 静默执行，不废话。用户说"准备那个东西"，AI 判断不了是什么 → 只问最关键的 1 个问题（"是写文档还是背材料？"），不是逐项盘问。置信度越高越安静，越低才越问——像真人搭档，不是填表机器。

### Part B：饮食管家

- **结构为主，卡路里为辅**：日常推荐看食物结构，隐形热量陷阱必须揭示（附带参照物）
- **三层分级冷启动**：只问2题（忌口+目标），其余自然时机获取或零打扰学习
- **目标-relative评分**：同一顿饭对不同目标评分不同
- **身体状况×忌口**：永久（过敏/脾胃弱）每餐自动避开，临时（嗓子疼/拉肚子）随口触发
- **偏好调整≠品类切换**：不想吃饭→只调优先级，不删品类
- **智能图片分析**：发食物图片→默认记录+评估+建议，推断意图不每次问
- **叙事式周报**：3句总结+3条行动项，永远正向框架
- **经期三阶段推算**：女性用户记录日期后自动推算周期，经前补铁防水肿→经期忌冰辣温热食→经后补血恢复，每个阶段推荐自动切换

#### 🌟 饮食两大核心亮点

**1. 隐形热量陷阱 — 参照物体系，不是裸数字**

"花生 170 卡"对大多数人没有意义。本 Skill 的陷阱检测不是甩一个数字，而是附带参照物："花生≈170卡≈半份米饭⚠️"。用户一眼就知道"这碟花生相当于我又吃了半碗饭"。参照物体系让热量陷阱变得可感知、可比较，而不是制造焦虑。

> 卡路里是工具不是目标。结构对了就不管数字，但陷阱必须揭示。这是 AI 智能设计的阈值策略，不是简单的"要不要算热量"开关。

**2. 三层分级冷启动 — 只问 2 题，其余全部靠推断学习**

传统饮食 App 第一次打开就问 7-10 题（身高体重性别预算运动口味作息过敏……）。本 Skill 只问 2 个安全必答题（忌口/过敏 + 目标），问完立刻给基线推荐，明确说"这是默认推荐，反馈后会更准"。其余信息按三层分级获取：
- 第二层：自然时机顺嘴问（用户说"没钱了"→ 自动省钱模式）
- 第三层：零打扰学习（口味从反馈记录推断，作息从排程数据导入）

> 不是"少问一点"，是 AI 真的有能力推断。能推断的不问，不能推断的才问——这是 AI 和传统 App 的根本区别。

### Part C：跨域联动（核心创新）

- **数据共享**：排程和饮食读写同一个 base.json，各自只改自己的字段
- **作息→饮食**：排程更新作息→饮食自动调整（夜猫→早午餐，早起→正常早餐）
- **饮食→排程**：减脂期→三餐排轻食，生理期→排程多留休息
- **模式切换联动**：排程切模式→AI主动问饮食要不要变
- **综合回应**："早上好"同时给出排程+三餐

#### 🌟 旗舰联动：经期排算系统 — 一个生理事件，两个域同时响应

女性用户记录经期日期后，AI 从历史记录中学习用户实际周期长度（首次记录暂用人群均值估计，≥2次记录后改用实际间隔推算），自动预测下次经期，并在经期前后**同时调整排程和饮食**：

| 阶段 | 饮食调整 | 排程调整 |
|------|---------|---------|
| **经前期**（来前3-5天） | 补铁红肉、减盐防水肿、维B6缓解情绪 | 正常排程，提醒"快来了注意别排太满" |
| **经期中**（3-7天） | **忌冰忌辣**，推温热汤类、红枣枸杞、温性食物 | **自动多留休息时间**，降低高强度任务密度，避免排满深度工作 |
| **经期后**（结束后7天） | 补血恢复：动物肝脏、红肉、深色蔬菜 | 逐步恢复正常排程强度 |

> 这不是两个功能各自加一个"经期模式"开关，而是 AI 识别到同一生理事件后，**同时**在两个域做出针对性调整——饮食忌口变了，排程强度也变了。传统 App 做不到跨域联动，更做不到按生理阶段自动切换。

## 适用人群

| 人群 | 痛点 | 本 Skill 怎么解决 |
|------|------|------------------|
| **大学生** | DDL多+拖延严重+食堂选择混乱+预算波动 | 拖延预拆解+倒推开工日+食堂推荐+月末省钱 |
| **考研/考证学生** | 固定复习节奏+长时间高压+饮食不能乱 | 能量时段排程+结构为主推荐+考试周情绪化进食检测 |
| **健身学生** | 减脂/增肌目标明确但食堂难选 | 目标-relative评分+陷阱检测+蛋白质缺口提醒 |
| **作息不规律的学生** | 夜猫/早起交替，三餐时间飘忽 | 作息→饮食联动：晚起推早午餐，早起推正常早餐 |
| **身体有忌口的学生** | 过敏/脾胃弱/经期忌口，食堂容易踩雷 | 永久忌口每餐自动避开+临时身体状况随口触发调整 |
| **女性学生** | 经期前后身体状态波动大，饮食和精力都需要调整 | 经期排算系统：自动推算周期→饮食三阶段切换（忌冰辣/补铁温热）+排程自动减负留休息 |

> 本 Skill 的核心设计假设是**学生场景**（食堂+DDL+作息浮动），但排程模块的动作推断和饮食模块的冷启动逻辑对任何有排程需求+饮食管理需求的人同样适用。

## AI 核心价值

> **没有 AI 这个功能根本做不到**：

1. **跨域联动推理** — 排程影响饮食、饮食影响排程，这种交叉推理传统 App 无法实现
2. **经期排算系统** — 一个生理事件同时触发两个域的调整（饮食忌口+排程减负），按三阶段自动切换，没有任何传统工具能做到这种性别感知的跨域联动
3. **语义推断** — 从"搞定简历"推断动作类型=写作+难度=3+时段=上午攻坚，不是关键词匹配
4. **三层分级冷启动** — 区分哪些信息必须问、哪些可以等、哪些不用问，传统 App 只能一次性问完
5. **置信度分叉** — AI 不确定时只问1个关键问题，不是逐项盘问
6. **拖延预拆解** — 连续3天移同任务→自动标记+拆解，这是 AI 从行为数据中发现的模式
7. **隐形热量陷阱** — 花生≈半份米饭，参照物体系是 AI 智能设计，不是裸数字

## 使用方式

### 环境准备

1. 安装 Hermes Agent (WorkBuddy)
2. 将 `skill/` 目录放入 `.workbuddy/skills/life-assistant/`
3. 饮食模块需设置环境变量：`GEMINI_API_KEY`（[免费获取](https://aistudio.google.com/apikey)）

### Skill 交互

在 WorkBuddy 对话中直接使用自然语言：

- **排程意图**："今天安排""有什么任务""DDL 快到了"
- **饮食意图**："吃什么""推荐午餐""记录饮食""发食物图片"
- **综合意图**："早上好""今天怎么安排" → 同时给出排程+三餐

### 脚本直接运行

```bash
# 排程引擎
python skill/scripts/scheduler.py --input data/sample_tasks_input.json --format table
python skill/scripts/scheduler.py --input data/sample_tasks_input.json --format json
python skill/scripts/scheduler.py --input data/sample_tasks_input.json --mode top3

# 拆解引擎
python skill/scripts/task_breakdown.py --input data/sample_tasks_input.json

# 饮食管家（需 GEMINI_API_KEY）
python skill/scripts/diet_manager.py recommend 示例用户
python skill/scripts/diet_manager.py record 示例用户 "番茄炒蛋+米饭"
python skill/scripts/diet_manager.py weekly 示例用户
```

## 技术特点

- **Python 标准库**：排程引擎和拆解引擎仅使用标准库（json/math/datetime），无需额外安装
- **免费 Gemini API**：饮食管家使用 Gemini 2.0 Flash（免费额度），实现多模态分析（文本+图片）
- **user×mode 分层存储**：支持多用户多模式，数据互不干扰
- **共享数据层**：排程和饮食共用 base.json，各自只改自己的字段，避免冲突

## 典型交互场景

### 场景1：首次使用 — 2题即推荐

```
用户：早上好，今天要写简历、背单词，中午吃什么

AI：第一次见面！先确认2个安全问题，马上给你推荐+排程：
1. 忌口/过敏/身体状况？[无] [海鲜] [花生] [酒精] [其他___]
2. 想怎么吃？[减脂] [增肌] [省钱] [随便]

用户：无，减脂

AI：收到！今天安排：
  11:00 起床+早午餐
  11:30-13:00 写简历（深度工作）
  14:00-15:30 考前冲刺背单词
  三餐建议：
  早午餐：番茄炒蛋+西兰花+半份米饭 ≈8元
  晚餐：白切鸡+清炒西兰花 ≈10元
⚠️ 这是基线推荐，反馈后会更准。
```

### 场景2：拖延克星 — 自动预拆解

```
用户：今天要完成论文

AI：这个任务你已经连续3天移到"今天"了 📊
我帮你拆成"穿上鞋"级别的步骤——
1. 打开文档，写5分钟就收工（5min）
2. 把大纲列出来（15min）
3. 填第一节内容（30min）
不需要一口气做完，先启动第一步就行。
```

### 场景3：隐形热量陷阱检测

```
用户：[发了午餐照片]

AI：记了！番茄炒蛋+米饭+一小碟花生米
减脂期 5分：蛋白质✓，蔬菜缺
⚠️ 花生≈170卡≈半份米饭——看着少，热量不少
晚餐：自选窗口加个清炒西兰花就补回来了
```

### 场景4：偏好调整 ≠ 品类切换

```
用户：不想吃饭，夏天汤汤水水好一点

AI：收到，把汤汤水水优先级调高，还是肉类午餐里挑：
1. 🥇 黄焖鸡汤（汤多饭少）
2. 🥈 小炒肉盖饭 + 配碗免费汤
3. 🥉 酸菜鱼米线
4. 烤肉饭（靠后排，想吃肉可以点）
5. 煲仔饭 → 不推荐，偏干偏腻
只想吃粉面的话再跟我说。
```

### 场景5：作息→饮食联动

```
用户：明天7点要去医院

AI：检测到明天7:00早起 → 今晚最晚0:30入睡
排程调整：22:00之后的任务挪到明天上午
饮食调整：明天早起→正常吃早餐（不是外卖），今晚晚餐推清淡早吃

好了告诉我，我自动切回正常模式。
```
