## 三、第一轮：作息采集缺失 + 排程脱离实际（7/12 上午）

**挂载版本：** v1.0 科目分类版
**改进时间：** 10:56 / 11:06

### 3.1 用户初始请求

> "今天需要完成两份作业，一份简单一份很难，中午需要收拾东西，今晚还要出去吃饭，帮我规划行程"

### 3.2 排程迭代（5轮才被接受）

| 轮次 | AI 排程 | 用户反馈 | 暴露问题 |
|:----:|:--------|:---------|:---------|
| V1 | 10:30简单作业→11:00继续→12:00收拾+午饭→13:00-16:00难作业→18:00晚饭 | "无法长时间深度思考，下午犯困" | 不考虑精力波动 |
| V2 | 难作业25min番茄钟→11:00出门→14:00午饭→15:00难作业 | "十点半吃午饭太早，空腹没法集中" | 不知道用户作息和三餐时间 |
| V3 | 提前吃午饭→午睡→难作业 | "这不是今天的，要明天完成" | 混淆今天/明天 |
| V4 | 8:00起床吃早餐→8:30难作业→10:30收拾 | "我早上睡到10点半，吃饭很慢要1小时" | 默认8:00起床，完全不问作息 |
| V5 ✅ | 10:30-11:30起床吃早餐→11:30难作业→12:00出门→15:00午睡→15:30难作业 | ✅ 接受 | — |

### 3.3 发现的核心问题（7项）

| # | 问题 | 严重度 |
|:-:|:----|:----:|
| 1 | **没有作息采集** — 不追问起床/睡觉/吃饭时间 | 🔴 |
| 2 | **默认用户已就绪** — 说"今天"就从当前时间开始排 | 🔴 |
| 3 | **三餐时间缺失** — 没有自动安排吃饭 | 🔴 |
| 4 | **任务跨度混淆** — 排了明天的任务当今天 | 🟡 |
| 5 | **不分阶段确认** — 一次性输出到18:00+ | 🟡 |
| 6 | **忽略精力波动** — 全程高强度安排 | 🟡 |
| 7 | **封装版保存旧科目分类** — 实际挂载的还是v1.0 | 🔴 |

### 3.4 触发的改进

1. 首次使用增加作息采集流程：问起床/睡觉/固定忙时/能量类型
2. 三餐按起床时间自动偏移推算（wake+30min→+4-5h→sleep-2-3h）
3. 增加 profile.yaml 存储作息/忙时/能量
4. 更新 SKILL.md 全面应用动作类型分析（替代旧科目分类）

## 四、第二轮：三餐弹性 + 排程紧凑度（7/12 11:11-11:20）

**挂载版本：** v2.0（含作息采集 + 任务池）

### 4.1 对话过程

用户 10:00 起床，AI 自动采集作息后排了两天搬家计划（周日打包+周一搬运）。

**用户反馈：**
> "太累了，我推迟两天回家"

**AI 调整：** 推两天，排程变宽松。

**用户又反馈：**
> "早餐呢，我的早餐嫩！"

**AI 补充：** "哈哈对不起，早餐忘了安排！10:00起床→10:00-10:30早餐，修正一下 👇"

### 4.2 发现的核心问题（3高+3中）

| # | 问题 | 严重度 |
|:-:|:----|:----:|
| 1 | 🔴 早餐默认缺失 — 10:00起床没自动加早餐 | 高 |
| 2 | 🔴 三餐不是按起床偏移推算 — 手动补的，规则没生效 | 高 |
| 3 | 🔴 作息采集了但没充分使用（早餐时间应为10:30不是10:00） | 高 |
| 4 | 🟡 排程太紧凑 — 没考虑"累"的状态 | 中 |
| 5 | 🟡 累了不会主动调整 | 中 |
| 6 | 🟡 任务量大时不会主动预警分摊 | 中 |

### 4.3 触发的改进

1. 三餐必须按起床时间自动推算，不是手动补
2. 排程增加弹性窗口（累/休息日）
3. 排程前先看任务量与天数比值，超载主动询问

## 五、第三轮：晚起处理 + 次日回顾 + 拖延预拆解（7/12-7/13 实际执行日）

**挂载版本：** v2.0

### 5.1 关键对话片段

**V1** — 用户按排程开始执行 Hadoop 任务，中途临时出门打印，16:26 回来提前完成。

**V2** — 用户说"有点久，提前半小时开始我得提前半小时结束"，AI 排程表仍按原计划到22:00没更新。

**V3** — 用户反馈"计划表怎么还是到22点呢"，AI 才修正。

**V4** — 次日14:30起床（晚了30min），计划一点都没变。

**V5** — 前一天跳过了一项任务没给AI反馈，第二天AI完全不问。

### 5.2 发现的核心问题（4项）

| # | 问题 | 严重度 |
|:-:|:----|:----:|
| 1 | 🔴 **提前完成未智能联动** — 排程表不随完成时间更新 | 高 |
| 2 | 🔴 **晚起不自动顺延** — 14:30起床但计划不变 | 高 |
| 3 | 🔴 **次日不主动回顾** — 跳过的任务第二天不问 | 高 |
| 4 | 🟡 **含糊反馈误判** — "hadoop任务完成"被当整体完成 | 中 |

### 5.3 触发的改进

1. 智能识别完成事件影响：提前完成→后续重排评估
2. 晚起触发自动顺延：偏移>15min→整体顺延+冲突校验
3. 主动状态回顾：第二天扫描昨日排程，逐项询问不明任务
4. **强化核心：拖延预拆解** — 同类任务未完成≥3次→下次自动预拆为子任务（事前预防，不是事后兜底）

## 六、第四轮：饮食管家合并 + 冷启动 + 饮食功能测试（7/13-7/14）

**挂载版本：** v4.0（合并排程+饮食联动版）

### 6.1 饮食推荐测试

**用户请求：** "中午吃什么"

**AI 推荐：** 番茄炒蛋+西兰花+半份米饭

**用户反馈：** "不想吃饭，夏天汤汤水水好一点"

**AI 调整：** 黄焖鸡汤排第1，小炒肉盖饭+配汤排第2，饭类靠后但保留。

> ❌ 初版错误：AI 直接把所有饭类推荐删掉了（品类切换而非优先级调整）。修正后改为只调优先级。

### 6.2 热量陷阱测试

**用户发图片 + 文字：** "番茄炒蛋+米饭+一小碟花生米"

**AI 检测：** ⚠️ 花生≈170卡≈半份米饭

> ✅ 参照物体系生效，不是裸数字。

### 6.3 临时身体状况测试

**用户说：** "嗓子疼不想吃辣"

**AI 调整：** 推荐蒸蛋+温粥+清炒时蔬≈6元，自动忌辣忌油炸

### 6.4 冷启动测试

**新用户首次使用：**

**AI：** "第一次见面！先确认2个安全问题：1.忌口/过敏？2.想怎么吃？"

**用户：** "无，减脂"

**AI：** 立刻给出排程+三餐推荐 + "⚠️ 这是基线推荐，反馈后会更准"

> ✅ 只问2题即推荐，不追问7个问题。

### 6.5 发现的核心问题（3项）

| # | 问题 | 严重度 |
|:-:|:----|:----:|
| 1 | 🔴 **偏好调整=品类切换** — "不想吃饭"删了所有饭类，应只调优先级 | 高 |
| 2 | 🔴 **冷启动问太多** — 初版问7题，用户信任感降低 | 高 |
| 3 | 🔴 **过敏/口味没问** — 初版把过敏放在"可选第二轮"，结果被跳过 | 高 |

### 6.6 触发的改进

1. 偏好调整≠品类切换：调优先级而非删品类，饭类靠后但不消失
2. 三层分级冷启动：只问2题（忌口+目标）→ 自然时机获取 → 零打扰学习
3. 过敏提升为必答第一层问题

## 八、脚本 CLI 测试

排程引擎和拆解引擎的脚本测试：

```bash
# 排程引擎 — table 格式
python skill/scripts/scheduler.py --input data/sample_tasks_input.json --format table
# ✅ 8个任务按优先级排列，自动拆解4个子任务

# 排程引擎 — JSON 格式
python skill/scripts/scheduler.py --input data/sample_tasks_input.json --format json
# ✅ 输出完整 JSON（stats + 8任务 + 深度/浅层时间统计）

# 排程引擎 — TOP3 聚焦
python skill/scripts/scheduler.py --input data/sample_tasks_input.json --mode top3
# ✅ 前3标记必做，其余延后

# 排程引擎 — 周视图
python skill/scripts/scheduler.py --input data/sample_tasks_input.json --format weekly
# ✅ 任务按日分配，DDL临近的排在截止日前一天

# 拆解引擎
python skill/scripts/task_breakdown.py --input data/sample_tasks_input.json --format text
# ✅ 拆解指数≥7自动拆、5-7待询问、<5跳过
```

> 详细输出见 `tests/output_scheduler_table.txt`、`output_scheduler_json.json`、`output_scheduler_top3.txt`、`output_scheduler_weekly.txt`、`output_breakdown.txt`、`output_breakdown.json`。
